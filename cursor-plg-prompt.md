# Cursor Prompt — Product-Led Growth (PLG) for BidSmith

**Mission: Bake PLG loops directly into the product. Every feature below must create a reason for users to invite others, share outputs, or return without being asked. No fluff — only mechanics that drive signups or revenue.**

---

## Context

BidSmith's core loop: paste SAM.gov URL or PDF → get bid/no-bid + compliance matrix → export XLSX.

PLG for BidSmith means: **the output itself becomes the acquisition channel.**

A govcon BD person who gets a compliance matrix from BidSmith should naturally share it with their team — and that share should pull the recipient into BidSmith.

---

## PLG Mechanic 1 — Branded shareable output

**What:** Every exported compliance matrix XLSX and any future PDF report gets a footer row:

```
Generated by BidSmith · bidsmith.pro/free-audit · Try it free →
```

**Where to implement:**
- `src/utils/complianceMatrixXlsx.js` — in `buildComplianceMatrixWorkbook`, after the last data row, add a styled footer row with the BidSmith attribution and URL
- Style it: light gray background (`EFEFEF`), italic, small font (9pt), hyperlink on `bidsmith.pro/free-audit`

**Why it works:** The file gets emailed internally. Every recipient who opens it sees the brand + CTA. No extra marketing spend.

---

## PLG Mechanic 2 — Shareable audit link (public read-only result)

**What:** After an audit completes, show a "Share this audit" button that generates a short public URL:

```
bidsmith.pro/audit/share/[token]
```

The shared page shows:
- Contract title + agency
- Bid/no-bid decision + score
- Top 3 compliance requirements (teaser)
- CTA: "See full matrix + export → Sign up free"

**Where to implement:**

Backend (`api/index.js`):
```js
// POST /api/audit/share
// Takes auditId, generates a signed read-only token, stores minimal result snapshot
// Returns { shareUrl }

// GET /api/audit/share/:token
// Returns the snapshot (title, decision, score, top 3 requirements)
// No auth required
```

Frontend:
- After `audit_success`, show `<ShareAuditButton auditId={id} />` next to the export button
- Clicking generates the share URL and copies it to clipboard
- Track: `track('audit_share_clicked', { audit_id })`

Shared page (`src/pages/SharedAudit.jsx`):
- Public route `/audit/share/:token`
- Shows teaser, hard CTA wall after requirement 3: "Sign up free to see all [N] requirements and export"
- Track: `track('shared_audit_view', { token, referrer })` and `track('shared_audit_signup_click', { token })`

**Why it works:** BD managers share audit results in Slack, email, Teams. Every view is a warm lead who already sees proof of value before signing up.

---

## PLG Mechanic 3 — Free tool as top-of-funnel (no signup required)

**What:** `/rfp-compliance-matrix-generator` — already in the route plan — must work with zero auth:

- User pastes a SAM.gov URL or uploads a PDF
- Gets bid/no-bid + first 3 compliance requirements **instantly, no signup**
- Full matrix + export is locked behind free account creation

**Where to implement:**
- Create a `FreeAuditGate` component: runs the audit API call without Clerk auth, renders the teaser result, shows signup modal on "See full matrix"
- Backend: add an unauthenticated rate-limited route `POST /api/audit/free` (limit: 1 per IP per 24h via Upstash or in-memory fallback)
- Track: `track('free_tool_audit_submitted')`, `track('free_tool_signup_prompted')`, `track('free_tool_converted')` (signup completed from this surface)

**Why it works:** Users experience the value before any friction. Conversion happens at the moment of "I want this full output" — highest intent moment possible.

## PLG Mechanic 4 — Invite team member (multiplier)

**What:** After export or audit success, show a one-line prompt:

```
"Collaborating on this bid? Invite your proposal manager →  [email input]  Send invite"
```

Clicking sends a templated email:
```
Subject: [Name] shared a BidSmith compliance matrix with you
Body: "[Name] ran [Contract Title] through BidSmith and wants you to review the compliance matrix. 
Click to see it and run your own: bidsmith.pro/free-audit"
```

**Where to implement:**
- Add `POST /api/invite` — takes `inviterName`, `inviteeEmail`, `contractTitle`, sends via your existing email setup (or Resend/SendGrid)
- Add `<InviteTeammate />` component in the post-audit panel, below the export button
- Track: `track('invite_sent', { source: 'post_audit' })`, `track('invite_accepted')` (on signup from invite link with `?ref=invite`)

**Why it works:** Proposal writing is a team sport. One user at a small GovCon firm pulls in their capture manager, PM, and BD lead — 1 signup becomes 3-4.

---

## PLG Mechanic 5 — SEO-indexed audit summaries (programmatic content)

**What:** Every public shared audit (Mechanic 2) that isn't marked private gets indexed:

```
bidsmith.pro/audit/[agency-slug]-[contract-slug]
```

Example: `bidsmith.pro/audit/dod-enterprise-cloud-migration-2024`

Page content:
- Contract title, agency, posted date, NAICS code
- Bid/no-bid summary (BidSmith's take)
- "Run your own analysis" CTA

**Where to implement:**
- In `SharedAudit.jsx`, add `<meta name="robots" content="index, follow">` when `isPublic: true`
- Add `<title>` + meta description dynamically from audit data
- Add to `sitemap.xml` generation (dynamic sitemap route `GET /api/sitemap` that lists public audits)

**Why it works:** Long-tail SEO. "NASA SEWP VI compliance matrix" — someone Googling that is an active bidder. You own that moment.
